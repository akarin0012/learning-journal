# IaC の概要を学んだ

- 日付: 2026-03-31
- カテゴリ: IaC / DevOps / Cloud

## 学んだこと

- IaC（Infrastructure as Code）は、サーバーやネットワークなどのインフラ構成をコードとして定義し、再現性と変更管理を高める考え方だと理解した。
- 手作業の設定と比べて、環境差分の削減、レビュー可能性の向上、構築の自動化がしやすくなる点が大きな利点だと学んだ。
- 宣言的に理想状態を記述する手法と、手順を記述する手法の違いがあり、ツールごとに思想が異なることを把握した。
- まずは小さな構成（例: 単一環境の基本リソース）からコード化し、状態管理と変更手順をチームで揃えることが実運用では重要だと感じた。
